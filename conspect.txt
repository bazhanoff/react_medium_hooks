    Арихтектура проекта (Гибкая структура, подходит для больших проектов)
  Файловая структура:
/node_modules
  /pages
    globalFeed.js
      /components // этим компоненты могу быть использованы внутри конкретной страницы, а не снаружи
        feed.js
    article.js
    authentication.js
  /components  // shareable components (переиспользуем между страницами)
    topbar.js
    pagination.js
    feed.js
  /hooks // наши кастомные хуки
    useFetch.js
  routes.js // наши роуты

Не хотим использовать относительные пути в наших импортах (Relative path), 
потому что выглядит некрасиво и сложно поддерживать
import Pagination from '../../../../components/pagination'
В CRA нет предпочтительного пути, как пофиксить эту проблему
Для решения этой проблемы создадим дополнительную папку node_modules и положим в неё все файлы нашего рпоекта.
Webpack сможет корректно понять, что мы хотим сделат, поскольку все наши файлы он будет 
считать одним node-модулем. Поэтому мы прямо из коробки получаем правильные абсолютные пути. 
Раньше этот вариант был одним из самых простых и был указан внутри CRA.
